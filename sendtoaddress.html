sendtoaddress - Particl RPC

sendtoaddress "address" amount ( "comment" "comment_to" subtractfeefromamount "narration" replaceable conf_target "estimate_mode" avoid_reuse fee_rate verbose )

Send an amount to a given address.
Requires wallet passphrase to be set with walletpassphrase call if wallet is encrypted.

Arguments:
1. address                  (string, required) The particl address to send to.
2. amount                   (numeric or string, required) The amount in PART to send. eg 0.1
3. comment                  (string, optional) A comment used to store what the transaction is for.
                            This is not part of the transaction, just kept in your wallet.
4. comment_to               (string, optional) A comment to store the name of the person or organization
                            to which you're sending the transaction. This is not part of the 
                            transaction, just kept in your wallet.
5. subtractfeefromamount    (boolean, optional, default=false) The fee will be deducted from the amount being sent.
                            The recipient will receive less particl than you enter in the amount field.
6. narration                (string, optional, default=) Up to 24 characters sent with the transaction.
                            Plaintext if sending to standard address type, encrypted when sending to stealthaddresses.
7. replaceable              (boolean, optional, default=wallet default) Allow this transaction to be replaced by a transaction with higher fees via BIP 125
8. conf_target              (numeric, optional, default=wallet -txconfirmtarget) Confirmation target in blocks
9. estimate_mode            (string, optional, default=unset) The fee estimate mode, must be one of (case insensitive):
                            "unset"
                            "economical"
                            "conservative"
10. avoid_reuse             (boolean, optional, default=true) (only available if avoid_reuse wallet flag is set) Avoid spending from dirty addresses; addresses are considered
                            dirty if they have previously been used in a transaction.
11. fee_rate                (numeric or string, optional, default=not set, fall back to wallet fee estimation) Specify a fee rate in sat/vB.
12. verbose                 (boolean, optional, default=false) If true, return extra information about the transaction.

Result (if verbose is not set or set to false):
"hex"    (string) The transaction id.

Result (if verbose is set to true):
{                          (json object)
  "txid" : "hex",          (string) The transaction id.
  "fee reason" : "str"     (string) The transaction fee reason.
}

Examples:

Send 0.1 PART
> particl-cli sendtoaddress "PswXnoragjpAtaySWkPSmWQe3Fc8LmviVc" 0.1

Send 0.1 PART with a confirmation target of 6 blocks in economical fee estimate mode using positional arguments
> particl-cli sendtoaddress "PswXnoragjpAtaySWkPSmWQe3Fc8LmviVc" 0.1 "donation" "sean's outpost" false true 6 economical

Send 0.1 PART with a fee rate of 1.1 sat/vB, subtract fee from amount, BIP125-replaceable, using positional arguments
> particl-cli sendtoaddress "PswXnoragjpAtaySWkPSmWQe3Fc8LmviVc" 0.1 "drinks" "room77" true true null "unset" null 1.1

Send 0.2 PART with a confirmation target of 6 blocks in economical fee estimate mode using named arguments
> particl-cli -named sendtoaddress address="PswXnoragjpAtaySWkPSmWQe3Fc8LmviVc" amount=0.2 conf_target=6 estimate_mode="economical"

Send 0.5 PART with a fee rate of 25 sat/vB using named arguments
> particl-cli -named sendtoaddress address="PswXnoragjpAtaySWkPSmWQe3Fc8LmviVc" amount=0.5 fee_rate=25
> particl-cli -named sendtoaddress address="PswXnoragjpAtaySWkPSmWQe3Fc8LmviVc" amount=0.5 fee_rate=25 subtractfeefromamount=false replaceable=true avoid_reuse=true comment="2 pizzas" comment_to="jeremy" verbose=true

Particl version: 0.21.2.6
